# PROJECT_MANIFESTO.md — DevContext

## Выбор проекта

После глубокого анализа трендов GitHub, Reddit, HackerNews и Stack Overflow за 2026 год, выявлена ключевая "боль" разработчиков:

**AI-ассистенты теряют контекст кодовой базы.**

Каждый разработчик тратит 10-15 минут за сессию объясняя AI-боту свой проект. При 5 сессиях в день = 1 час потерь ежедневно = 365 часов в год.

## Почему DevContext побеждает

| Критерий | DevContext | Конкуренты |
|----------|------------|------------|
| Простота | `pip install` + `devcontext generate .` | Сложные API-ключи, настройки |
| Стоимость | Бесплатно | Платно или фримиум с лимитами |
| Языки | 20+ | 3-5 обычно |
| Офлайн | ✅ | ❌ |
| Open Source | ✅ | ❌ |

## Целевая аудитория

- **Основная:** 50M+ разработчиков в мире, использующих AI-ассистентов
- **Вторичная:** Команды, переходящие на AI-first development

## Виральная механика

1. **Wow-эффект:** "О, теперь AI понимает мой проект за 3 секунды!"
2. **Социальное доказательство:** Звезды на GitHub = доверие
3. **Полезность:** Решает реальную проблему, не созданную

## Финализация

**Проект:** DevContext — CLI для генерации AI-ready контекста из кодовой базы.

**Обоснование:**
- ✅ Массовая аудитория: все разработчики
- ✅ Решает реальную проблему
- ✅ Технически реализуемо автономно
- ✅ Бесплатно и open source
- ✅ Быстрая установка (pip install)
- ✅ Единая точка входа (CLI)

**Статус:** ЗАФИКСИРОВАНО. Проект неизменен до достижения 5000 звёзд.
